LPS / PS-LPS Focused Design-Basis Backend Comparison

Generated: 2026-06-06 22:28:43 EDT

Purpose

This focused backend comparison checks whether the new guarded local polynomial design backends behave sensibly on a small set of frozen non-manifold fixtures. The comparison is intentionally small: it is a backend audit exercise, not a performance claim about final model defaults.

The synthetic target is Truth RMSE,

TruthRMSE = sqrt{ n^{-1} sum_i (hat f_i - f_i)^2 }.

Observed CV RMSE is used only as the non-oracle selection score inside each method; Truth RMSE is used after fitting to audit how well the selected fit recovered the known synthetic function.

Methods

The run uses deterministic subsamples of four frozen first-batch assets: LA-D1-RAW-N500.rds, LA-D1-HC-Li-N500.rds, SYN-PARA-LINE-N500.rds, SYN-RANK-BLOCKS-N600-P100.rds. Each fit uses chart.dim = "auto", local PCA coordinates, degree 2, tricube kernel, and support grid {15, 25}. PS-LPS uses lambda.sync.grid = {0, 0.01, 0.1} with grid search and the normal-cache solver.

The backend variants are:

  • monomial_tiny_ridge: standard monomial design with fixed tiny scale-relative ridge.
  • weighted_qr_drop_tiny: weighted-QR rank/drop guard followed by zero or tiny ridge subject to a condition cap.
  • orthogonal_drop_ridge0: orthogonalized local polynomial basis with dropped rank-deficient columns and no ridge.
  • orthogonal_drop_adaptive_tiny: the same orthogonal basis plus zero-or-tiny adaptive ridge subject to a condition cap.

Interpretation

This run asks whether the backend choices are numerically explicit and auditable. A successful result is not necessarily the lowest Truth RMSE; it is also important that failures are reported as failures, rank drops are visible, and ridge choices are recorded.

The orthogonal basis with zero ridge should be interpreted as a change of basis for the same local polynomial prediction space after rank-deficient directions are dropped. Once a ridge is added, the penalty is applied in the orthogonalized coefficient coordinates, so it is not algebraically identical to ridge on raw monomial coefficients. For prediction this is usually the more stable numerical target, but the distinction should be audited before promoting defaults.

Failures or non-finite fits were observed. See tables/lps_ps_lps_design_basis_failures.csv.
