## Goal

Run the coupled support-size and chart-dimension selection experiments in the
degree-2 local-polynomial regime.  The motivating issue is that CSD5--CSD9 were
previously run with local polynomial degree `g = 1`, while the target LPS-like
model regime is degree `g = 2`.

The degree-2 lane is labeled `CSD-deg2`.

## Work Completed Before Launch

The existing CSD runners were modified so that the degree is an explicit
runtime parameter instead of a hard-coded `1L`.

The changed scripts are:

- `dev/methods/lps/ci/csd5_coupled_kd_evaluation_run.R`
- `dev/methods/lps/ci/csd6_expanded_relative_regret_run.R`
- `dev/methods/lps/ci/csd8_candidate_cv_surface_audit_run.R`
- `dev/methods/lps/ci/csd7_task_failure_diagnostics_render.R`
- `dev/methods/lps/ci/csd9_robust_cv_selection_policy_audit_render.R`

The CSD5, CSD6, and CSD8 runners now accept:

```sh
--degree=2
--report-dir=<output-directory>
```

The CSD7 and CSD9 renderers were made degree-aware in their boundary diagnostics.
They no longer hard-code chart-dimension boundary `d in {1, 8}`.  Instead, they
derive the support and chart-dimension boundary from the feasible candidate grid
present in the input tables.

## Limitations And Unverified Claims

The generated reports and summary tables exist, but they have not been
independently audited.  The numerical findings above were copied from generated
CSV summaries, not recomputed independently from raw per-candidate rows.

The report titles were patched to identify `CSD-deg2`, but some body prose still
inherits wording from the degree-1 CSD reports.  The outputs are interpretable,
but a later report-polish pass may still be useful.

The first failed wrapper launch was an orchestration error, not a model-fitting
error.  It produced no scientific artifacts.  The accidental outside-repo
directory was removed.
