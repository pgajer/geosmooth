## Exported generators and their bindings

Each generator implements the plan `sec:dgp` definition exactly and returns a
`dgp_dataset`. "Bound to" names the consolidated asset(s) the implementation
draws on (vendored/adapted, not forked).

| G-tag | Exported fn | Plan `sec:dgp` definition implemented | Bound to |
|---|---|---|---|
| G1 | `dgp.g1` | ambient `Unif([-1,1]^D)`; default `D=2,p=2` truth `0.5+1.0x1-0.7x2+0.4x1^2+0.3x1x2-0.6x2^2`; noiseless unless `sigma>0` | `make.flat.dataset` (ssrhe) + plan polynomial |
| G2 | `dgp.g2` | latent `Unif([-1,1]^d)`; fixed random orthonormal `Q` (recorded); `X=Q u`; degree-`p` truth in `u` | flat helper + orthonormal frame |
| G3a | `dgp.g3a` | disk radius `rho0=1`; `X=(u1,u2,(u1^2+u2^2)/(2R))`; apex `kappa=1/R`; truth `u1` or `sin(pi u1)cos(pi u2)` | `quadform.embed` (gflow) + geosmooth `2d_curved_paraboloid` |
| G3b | `dgp.g3b` | disk `‖u‖<=rho0<R`; `X=(u1,u2,R-sqrt(R^2-‖u‖^2))`; Gaussian curvature `1/R^2` | explicit sphere cap (plan) |
| G3c | `dgp.g3c` | helix `X(t)=(cos t,sin t,c t)`, `t~Unif[0,2pi)` | explicit helix (plan) |
| G3d | `dgp.g3d` | torus radii `(R1,R2)`; latent angles on a sub-rectangle; standard torus embedding | explicit torus patch (plan); `2d_curved_saddle` is geosmooth provenance |
| G4 | `dgp.g4` | stratum A 1-D segment `{(t,0,0):t∈[-1,0]}` thickened by `eta`; stratum B 2-D patch `{(s1,s2,0)}`; region labels A/B | explicit glued strata (plan); SYN-TWO-PLANES family is the FB provenance |
| G5 | `dgp.g5` | `K` clusters, centres `Unif([-1,1]^2)`; `x=c_k+N(0,sigma_x^2 I)`; `y=g(x)+b_k+eps`, ICC `rho=tau^2/(tau^2+sigma^2)` | SYN-DISK-CLUSTERS family + plan repeated-measures model |
| G6 | `dgp.g6` | `eta(x)=1.5 sin(pi x1)`; `alpha` solved so mean expit = target prevalence; `p=clip(expit(alpha+eta),0.05,0.95)`; `y~Bernoulli(p)` | `probability.profile` (lps_binary_gm_ff_helpers) reduced to plan G6 |
| G7 | `dgp.g7` | `D`-part composition `Dirichlet(alpha)`; a documented subset of parts set to exact 0; renormalize | plan Dirichlet-with-zeros; LA-*/SYN-SIMPLEX-FACES are the FB provenance |

### Standard dataset object (`.dgp.dataset`)

Fields: `dataset.id`, `gtag`, `geometry.family`, `n`, `p` (ambient dim),
`d` (intrinsic dim, `NA` when it varies by region), `U` (intrinsic/latent),
`Z` (documented alias of `U`), `X` (observed), `truth` (noiseless `f`; for G6
the probability `p`), `y` (noisy response; for G6 the Bernoulli draw), `sigma`
(`NA` for binary G6), `region` (labels or `NULL`), `seed`, `params`,
`provenance`. Class `"dgp_dataset"` with a `print` method.

### RNG discipline (spec `sec:rng`)

Each generator draws its geometry/latent immediately after `set.seed(seed)` and
its response noise immediately after `set.seed(seed + 900000L)`; the G2 frame
uses `set.seed(seed + 500000L)` (offsets are module constants). Each generator
is therefore a pure function of its arguments. Replicate `r` of a study is
`seed = s0 + r` (the caller's responsibility; not baked into the generators).

## Limitations and unverified claims

- **Plan definition vs. FB asset bindings.** The exported generators implement
  the plan `sec:dgp` synthetic definitions. The shared brief's binding table
  additionally associates some tags with frozen non-manifold/real-data assets
  (G4→FB10–FB14, G6→the binary factorial manifest, G7→LA-* FB01–FB09 +
  SYN-SIMPLEX-FACES FB13 + LA-13K-SUB). This module does **not** regenerate the
  VALENCIA-derived real-data LA-* assets (they require the VALENCIA 13k matrix
  and the `linf` embedding pipeline, which are not materialized here), nor does
  it reproduce the FB synthetic registries; `dgp.g7` is the plan's
  Dirichlet-with-structural-zeros DGP, not a re-materialization of FB13/LA-*.
- **Truth functions not pinned by the plan.** The plan fixes no truth for G3c,
  G4, and G7. I used documented defaults (`sin(t)`; `sin(pi*x1)`;
  `sin(pi*p1)+0.5*p2`). G5's mean `g` and G4's truth are also defaults. These
  are implementation choices, configurable via arguments.
- **G3d is the torus only.** The plan defines G3d as a torus patch; that is what
  `dgp.g3d` implements. A saddle variant (the `2d_curved_saddle` provenance) is
  not exposed as G3d.
- **G1/G2 polynomial coverage.** Only `D=2, p=2` (G1) and `d=2` (G2) carry the
  plan-pinned polynomial; other dimensions require an explicit `poly.coef` and
  otherwise raise an error (no silent default).
- **G6 offset semantics.** `alpha` is solved so the mean *unclipped* expit equals
  the target (the plan's literal wording); the realized *clipped* mean differs
  slightly (recorded in `params$realized.prevalence`), and the realized Bernoulli
  rate `mean(y)` differs further by sampling. The test asserts the clipped mean
  within `0.03` of target, not exact equality.
- **Curvature verification scope.** For G3a the apex principal curvature `1/R`
  is checked numerically and algebraically. For G3b/G3c/G3d only the on-surface
  membership identities (sphere/cylinder/torus) and the analytic curvature values
  recorded in `params` are asserted; the Gaussian/principal curvatures of G3b and
  G3d are **not** independently re-derived from a full second-fundamental-form
  computation in the test.
- **Checksum portability.** `dgp.content.sha256` uses `digest`'s R serialization.
  Determinism is verified within the frozen environment (R 4.5.2, digest 0.6.39);
  bitwise reproducibility across other R / serialization-format versions is not
  established. The manifest pins the environment.
- **Registry scope.** The registry freezes 24 canonical seed=1 (replicate-0)
  instances tied to the waiting studies; it does not enumerate the full
  replicate sweeps `seed=s0+r` that studies will draw on demand.
- **Evidence channel.** The testthat run was executed in the interactive
  implementer session via `pkgload::load_all`, not through
  `scripts/ci/run_tier0_execution_artifact.sh` or CI; no committed
  `dev/methods/lps/audit_artifacts/` bundle exists (`dev/methods/lps/audit_artifacts/` is gitignored). The
  committed evidence is `inst/dgp_registry/` (registry, manifest, sessionInfo).
- **Mutation / falsification not run.** Per authorship independence, I did not
  run any mutation, falsification, or independent re-derivation of these
  generators against the plan.
